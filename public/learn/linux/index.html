LINUX

You already live here. This guide sharpens the muscle memory and hands you better tools.

Sahil just got a key. Not a metal one, an SSH key, and behind it sat a small Linux box humming in a closet somewhere, freshly installed, empty, waiting. No graphical desktop. Just a blinking cursor and a prompt that ended in a dollar sign.

That box is the through-line of this guide. Every command below is something Sahil actually needed: to move around, read a file, find a config, fix a permission, kill a runaway process, install a package, reach another machine. We keep the theory thin and the examples real.

If you have used Linux for years, treat this as a refresher with a sharper toolbox bolted on. If you are newer, follow Sahil step by step. Either way, the goal is the same: by the end you should be able to sit at a strange machine and feel at home.

Orientation
01

The Lay
of the Land

First instinct in a new place: figure out where you are standing and what is around you. Three commands answer that, and Sahil types them in the same order almost every time.

sahil@box: ~
$ pwd                # print working directory: where am I?
/home/sahil
$ ls                 # list what's here
$ ls -lah            # long view, all files, human sizes
$ cd /etc            # change directory (absolute path)
$ cd ..              # up one level
$ cd -               # jump back to the previous directory
$ cd                 # home, from anywhere

Two ideas do most of the work here. A path starting with / is absolute (measured from root); anything else is relative to where you stand. And the trio . (here), .. (up), and ~ (home) appears everywhere, so it is worth burning in early.

Tip. ls has too many flags to memorize. Sahil keeps just three: -l (details), -a (hidden dotfiles), -h (human-readable sizes). Mashed together: ls -lah.

Seeing the tree

To see structure rather than one flat list, tree draws the whole branch. On a big directory it is overwhelming, so cap the depth.

sahil@box: ~/project
$ tree -L 2          # two levels deep, no deeper
$ eza --tree --level=2   # the modern, colorful version eza > tree
Sahil's first upgrade: eza replaces both ls and tree with color, icons, and a built-in tree mode. We will gather all such swaps in the Sharper Tools section.
Reading / Writing
02

Reading &
Writing Files

Sahil needs to peek at a config file, skim a long log, and create a few files and folders. This is the daily bread of working in a shell.

Looking inside

sahil@box: /etc
$ cat hosts          # dump a whole (small) file
$ less syslog        # scroll a big file; q to quit, / to search
$ head -n 20 log     # first 20 lines
$ tail -n 20 log     # last 20 lines
$ tail -f log        # follow live, as new lines arrive
$ bat hosts          # cat with syntax highlight + line numbers bat > cat
The one to remember: tail -f. When something is broken and writing to a log, this lets Sahil watch it happen in real time.

Making and moving

sahil@box: ~
$ mkdir -p a/b/c     # create nested dirs in one shot
$ touch notes.txt    # create an empty file (or bump its timestamp)
$ cp src.txt dst.txt # copy
$ cp -r dir1 dir2    # copy a directory (recursive)
$ mv old.txt new.txt # rename OR move (same command)
$ rm notes.txt       # delete a file
$ rm -r dir2         # delete a directory and its contents
Caution. There is no trash can. rm -rf deletes recursively with no questions asked, and it does not come back. Sahil reads the path twice before pressing Enter, especially after sudo.

Editing in place

Two editors live on nearly every machine. nano is gentle (shortcuts shown at the bottom; Ctrl+O saves, Ctrl+X exits). vim is powerful and modal; the survival fact is that Esc then :wq writes and quits, and :q! quits without saving.

Search
03

Finding
Things

A week in, Sahil's box has thousands of files. The skill that separates the comfortable from the frustrated is finding things fast: by name, or by what they contain.

By name

sahil@box: ~
$ find . -name "*.conf"        # classic: search by filename
$ find . -type f -mtime -1     # files changed in the last day
$ fd conf                      # same idea, faster + simpler syntax fd > find
$ fd -e conf                   # by extension
find is everywhere and worth knowing, but its syntax is fiddly. fd covers the common 90% with a fraction of the typing and respects your .gitignore by default.

By content

sahil@box: ~/project
$ grep "timeout" config.yml    # find a word in a file
$ grep -rn "TODO" .            # recursive, with line numbers
$ rg "TODO"                    # ripgrep: recursive by default, very fast rg > grep
$ rg -i "error" -g "*.log"     # case-insensitive, only .log files
Sahil's verdict: ripgrep (rg) is the single biggest quality-of-life jump on this list. It searches an entire project in milliseconds and skips junk directories automatically.

Fuzzy, interactive

When Sahil half-remembers a name, fzf turns any list into a live, type-to-filter picker. Piped together with the tools above it becomes a search cockpit.

sahil@box: ~
$ fd -e md | fzf               # pick a markdown file interactively
$ vim "$(fzf)"                 # fuzzy-pick a file, open it in vim
Access
04

Permissions
& Ownership

Sahil tries to run a script and Linux says Permission denied. This is the moment everyone meets the permission model, and once you can read that string of letters, it stops being mysterious.

Run ls -l and the first column is exactly this string. Three groups of three: what the owner may do, what the group may do, what others may do. Each slot is read (r), write (w), or execute (x).

sahil@box: ~/scripts
$ chmod +x deploy.sh   # make it executable
$ chmod 755 deploy.sh  # owner rwx, group/other r-x
$ chmod 644 notes.txt  # owner rw, everyone else read
$ chown sahil file     # change owner
$ chown sahil:dev file # change owner and group

And then there is sudo: run one command as the superuser (root), the account that can override these rules. It is the difference between a request being denied and being obeyed. Use it deliberately.

Mental shortcut. 755 for things you run, 644 for things you only read or edit. That pair covers most of daily life.
05

Who
You Are

Permissions only make sense once you know who the system thinks you are. A few quick commands answer that, and they are the first thing Sahil checks when something behaves unexpectedly.

sahil@box: ~
$ whoami             # your username right now
$ id                 # your user id, group ids, all groups
$ groups             # which groups you belong to
$ sudo -i            # become root for a session (careful)
$ su - otheruser     # switch to another user
Group membership is why sudo works for you and not for everyone: being in the sudo (or wheel) group is what grants the privilege. id shows it.

Creating users (useradd, passwd) is admin territory you will reach for less often. Knowing it exists, and that id and groups explain most permission surprises, is enough to keep moving.

Runtime
06

Processes &
Performance

Something is eating the CPU and the box feels sluggish. Sahil needs to see what is running and, if necessary, stop it. Every program you launch becomes a process with a number, its PID.

sahil@box: ~
$ ps aux             # snapshot of every running process
$ ps aux | grep node # find a specific one
$ top                # live, sorted by CPU; q to quit
$ htop               # top, but readable and scrollable htop > top
$ btop               # prettier still, with mouse + graphs btop > htop
$ kill 4821          # ask process 4821 to stop
$ kill -9 4821       # force it (last resort)

Foreground, background, jobs

A long task does not have to block the terminal. Sahil can suspend it, push it to the background, and bring it back.

sahil@box: ~
$ long-task &         # start it in the background
$ jobs               # list backgrounded jobs
$ fg %1              # pull job 1 back to the foreground
# Ctrl+Z suspends the current task; bg resumes it in the background
Reach for kill -9 last. A plain kill lets a program shut down cleanly. The -9 form yanks the plug, which can leave half-written files behind.
Software
07

Packages &
Software

Sahil needs to install a tool. On Linux you rarely download an installer; instead a package manager fetches software (and its dependencies) from trusted repositories. Which one you use depends on the distro family.

Debian / Ubuntu  (apt)
$ sudo apt update            # refresh the package list
$ sudo apt install ripgrep   # install
$ sudo apt remove ripgrep    # uninstall
$ apt search fzf             # find a package
Fedora / RHEL (dnf)  ·  Arch (pacman)
$ sudo dnf install ripgrep   # Fedora family
$ sudo pacman -S ripgrep     # Arch family (-S = sync/install)
$ paru -S yazi               # Arch + AUR helper for community pkgs
The pattern is always the same: update the list, then install by name. Only the verb changes between distros. Learn your machine's manager and the rest translate at a glance.
Connectivity
08

The
Network

The box does not live alone. Sahil checks its address, tests whether it can reach the outside world, fetches a file, and connects to another machine.

sahil@box: ~
$ ip a               # this machine's network addresses
$ ping example.com   # can I reach it? Ctrl+C to stop
$ ss -tulpn          # which ports am I listening on?
$ curl -s url        # fetch a URL (great for APIs)
$ wget url           # download a file to disk

Reaching another machine

sahil@laptop: ~
$ ssh sahil@server          # open a remote shell
$ scp file.txt sahil@server:~  # copy a file over there
$ ssh-keygen -t ed25519     # make a key pair (no more passwords)
This is how Sahil reached the box in the first paragraph. SSH keys are the standard: generate a pair once, put the public half on the server, and you log in without typing a password again.
Daemons
09

Services
& Logs

Long-running programs (a web server, a database, SSH itself) are managed as services by systemd. When something should be running but is not, this is where Sahil looks.

sahil@box: ~
$ systemctl status nginx     # is it running? why not?
$ sudo systemctl start nginx # start it now
$ sudo systemctl enable nginx# start it on every boot
$ sudo systemctl restart nginx
$ journalctl -u nginx -f     # follow this service's logs live
The debugging loop: status tells you the state, journalctl -u <svc> -f shows you why. Together they answer most "it stopped working" questions without guessing.
Composition
10

Pipes, Power
& the Shell

This is where Linux stops being a list of commands and starts being a language. The shell lets Sahil connect small tools into one custom tool, on the spot. The idea is plumbing: the output of one command becomes the input of the next.

sahil@box: ~
$ cmd1 | cmd2        # pipe: send output of cmd1 into cmd2
$ cmd > out.txt      # redirect output into a file (overwrite)
$ cmd >> out.txt     # append instead
$ cmd1 && cmd2       # run cmd2 only if cmd1 succeeded
$ cmd1 || cmd2       # run cmd2 only if cmd1 failed
$ fd -e log | xargs rm   # feed a list as arguments to another cmd

Making it yours

Anything Sahil types often becomes an alias, saved in the shell's config file (~/.bashrc or ~/.zshrc) so it loads every session.

~/.zshrc
# shortcuts that stick around
alias ll='eza -lah'
alias gs='git status'
# a variable, and using it
export EDITOR=vim
The leap worth making: switch from bash to zsh with a prompt like starship. You keep every command in this guide, and gain autosuggestions, smarter completion, and a prompt that tells you what you need at a glance.
11

Sharper
Tools

Everything above works on a bare machine, and you should know it cold. But Sahil does not stop there. These modern replacements are faster, friendlier, and (mostly) drop-in. Install the few that fit how you work; do not install all of them at once.

ripgrep (rg)

replaces grep

Searches file contents across a whole project in milliseconds. Skips ignored and binary files automatically.

fd

replaces find

Find files by name with simple syntax and sane defaults. Respects .gitignore.

bat

replaces cat

Prints files with syntax highlighting, line numbers, and git markers. Pages long files for you.

eza

replaces ls / tree

Colorful listings, icons, and a built-in --tree mode. Friendlier ls -lah.

zoxide

replaces cd

Learns your most-used directories. Type z proj and jump there from anywhere.

fzf

adds: fuzzy find

Turns any list into a live, type-to-filter picker. Pipe anything into it.

btop

replaces top / htop

A live system monitor with graphs, mouse support, and a layout that is actually readable.

dust

replaces du

Shows what is eating your disk as an instant, sorted visual tree.

tldr

replaces man (quick)

Community cheat-sheets. tldr tar gives the five examples you actually wanted.

yazi

adds: file manager

A blazing terminal file manager with previews. Browse, rename, and move with the keyboard.

zsh + starship

replaces bash prompt

A better shell and a fast, informative prompt. Autosuggestions and git status, built in.

ghostty

terminal emulator

A fast, GPU-accelerated terminal with sane defaults. The window Sahil actually types into.

hyprland

tiling window manager

For when the desktop itself becomes keyboard-driven. Tiling, scriptable, Wayland-native.

delta

improves git diff

Side-by-side, syntax-highlighted git diffs that are genuinely pleasant to read.

12

Field
Test

Answer out loud before you open each one. If you can explain it to Sahil, you know it.

Q1What does rwxr-xr-- mean, and what is it as a number?+
Owner has read+write+execute (7), group has read+execute (5), others have read only (4). So it is 754. Read=4, write=2, execute=1, added per group of three.
Q2You need to watch a log file as new lines appear. Which command?+
tail -f /path/to/log. The -f means "follow". For a service under systemd, journalctl -u <name> -f does the same.
Q3What is the difference between > and >>?+
> overwrites the file with the command's output. >> appends to the end, keeping what was already there.
Q4A process with PID 9001 is stuck. What is the difference between kill 9001 and kill -9 9001?+
Plain kill sends a polite signal asking the process to shut down cleanly. kill -9 forces termination immediately with no cleanup. Try the polite one first.
Q5Read this aloud: make && ./run || echo failed+
Run make. If it succeeds, run ./run. If either of those fails, print "failed". && chains on success, || chains on failure.
Q6You want every file containing the word "timeout" under the current folder. Bare tools, then the sharp one.+
Bare: grep -rn "timeout" .   Sharp: rg "timeout" (recursive by default, faster, skips junk).
13

Hands
On

Sahil's box needs tidying. Do these on a real machine (or a throwaway container). Hints point you at the right tool; the exact incantation is yours to assemble. That is the part that builds memory.

  1. From your home directory, create a folder lab/ with three nested subfolders inside it, using a single command. Hint: one mkdir flag turns a path into all the directories along it.
  2. Create an empty file lab/log.txt, then make it readable and writable by you but read-only for everyone else. Hint: touch, then a three-digit chmod. Recall the number for "owner read+write, others read".
  3. Find every .conf file under /etc and count how many there are. Hint: fd (or find) produces the list; pipe it into wc -l to count lines.
  4. Search all of /etc for any file containing your username, ignoring case, without drowning in permission-denied noise. Hint: rg with a case-insensitive flag; it already skips what it cannot read.
  5. Start a process that just waits (for example sleep 300) in the background, confirm it is running, then stop it by its PID. Hint: trailing &, then jobs or ps, then kill.
  6. In one line, take a log file, keep only the lines containing "error", and write them to errors.txt. Hint: grep (or rg) piped or redirected with >.
  7. Add one alias to your shell config that you would genuinely use, then reload the config without opening a new terminal. Hint: edit ~/.bashrc or ~/.zshrc, then source it.
  8. Stretch: install one tool from the Sharper Tools section with your package manager, and replace one habit with it for a day. Hint: rg or fd are the easiest wins to start with.
