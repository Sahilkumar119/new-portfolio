The schema you'll use throughout

Every module runs on the same tiny company database, so you focus on technique, not on re-reading a new schema each time. People work in departments, some manage others, and they place orders.

departments
dept_id (PK) · dept_name · location
employees
emp_id (PK) · name · dept_id (FK→departments) · manager_id (FK→employees, self-ref) · salary · hire_date
orders
order_id (PK) · emp_id (FK→employees) · customer · amount · order_date · status
